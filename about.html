Welcome to my google summer of code Project 2016

A configurable BioJS visualization component used to analyze and discover relationships in data.

Live demo

Project purpose

The concept of this visualization component is to create summaries of information regarding biological data and ideally serve as a multiprupose tool for data analysis with similar complex structure.

The above pie chart attempts to act as a filter in copmplicated data structures containing multiple types (in our case diseases, protein-targets, shared-phenotypes, drugs). And provide a simple but effective way of analysis.

In the initial state the visualization provides an overview of the differnet data types existing in a data structure.

Once clicked a user sees all the underlying data of a specific type.

The data points depending on their value each which may signifies different the distance from the center will be allocated accordingly. This way with an instant look the user will know data with the highest values and will be able to have an astimation of the ditribution of the values of the data.

The five rings seperate intenionally the data into 5 equal pieces depending on their values. By clicking on the bull rings transition and the clicked one expands to the maximum acting as a filter and focusing on the piece of the selected ditributino.

Lastly the user may click in a specific bullet or label to learn more about an individual object through a small versatile tooltip.

The github repository of the web component is located here with analytical instructions on its use while it has already been uploaded in the npm and biojs registries accordingly.

Experiments & future plans

In the beginning we had experimented with graphs during our effort to find intuitive ways to analyze realtionships of data. As a result quite some prototypes were made in the process. One of the future goals of the project is to combine the two ideas and create graphs where the nodes are somewhat a more complicated entity than a simple bubble.

Using pies in the positions of multiple nodes would greatly reduce the complexity of complicated graphs, so the experiments done are not to be abandonned. For the purpose of this evaluation they were put in this repository temporarilly.
