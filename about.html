Welcome to my google summer of code Project 2016

A configurable BioJS visualization component used to analyze and discover relationships in data.

Live demo

Project purpose

The concept of this visualization component is to create summaries of information regarding biological data and ideally serve as a multiprupose tool for data analysis with similar complex structure.

The above pie chart attempts to act as a filter in copmplicated data structures containing multiple types (in our case diseases, protein-targets, shared-phenotypes, drugs). And provide a simple but effective way of analysis.

In the initial state the visualization provides an overview of the differnet data types existing in a data structure.

Once a pie piece is clicked a user sees all the underlying data of a specific type.

Return to the initial stateThe happens via the "Home".

The data points depending on their value, which may have different significance, are located in proportionate distances from the center according to that value. The topology permit the end user to acquire an immediate estimation of the distribution of among the dataset.

The five rings seperate the circle into five equal pieces trying to accomplish visually a better distinction among data. . By clicking on any of the bull rings, they transition and the clicked one expands until the edge of the circle pushing the dat points accordingly, acting as a filter and focusing on a specific "ring".

Lastly the user may click in a specific bullet or label to learn more about an individual object through a small versatile tooltip.

The github repository of the web component is located here with analytical instructions on its use while it has already been uploaded in the npm and biojs registries accordingly.

Experiments & future plans

In the beginning we had experimented with graphs during our effort to find intuitive ways to analyze realtionships of data. The prototypes that were made during this process will be used in to combine the graph module with the pie chart one in the future. In this case the graphs will contain nodes which are somewhat a more complicated entity.

Using a pieChart instead of multiple nodes would greatly reduce the complexity of heavy loaded graphs. Although the graph prototype is in primitive state, for the purpose of this evaluation they were put in this repository temporarilly.
